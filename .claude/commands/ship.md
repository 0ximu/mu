# /ship - Commit and PR Workflow

## Steps

1. Run `git status` and `git diff` to review changes
2. Create commit with `[skip ci]` in message (for draft PRs)
3. Push to branch
4. Add PR comment with concise summary of changes

## Commit Message Format

```
feat: [brief description]

[skip ci]
```

## PR Comment Format

```markdown
## Changes

- [Bullet point summary of changes]
- [Another change]

## Testing

- [How changes were tested]
```
